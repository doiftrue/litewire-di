your-plugin.php PHP 
use Kama\LiteWireDI\Container;

final class Logger {
	public function log( string $message ): void {
		error_log( $message );
	}
}

final class Service {
	public function __construct(
		private Logger $logger
	) {}
}

$container = new Container();

// Logger is wired automatically.
$service = $container->get( Service::class );

Small by design

Keep the container out of your way.

LiteWire DI is made for plugins, themes, libraries, and focused PHP apps where a framework-scale container is more machinery than value.

Automatic wiring

Constructor and factory dependencies resolve from their type declarations.

02

Shared or fresh

Use get() for shared services and make() for a new root object with runtime values.

03

Strict object model

Class and interface names are service IDs. No magic strings or mixed-value storage.

04

Factories included

Register objects, class names, or closures. Factory dependencies are autowired too.

05

Useful failures

Circular dependencies are detected and reported with the full resolution chain.

06

PSR-11-style API

Familiar get() and has() methods without adding psr/container as a dependency.

The whole API
